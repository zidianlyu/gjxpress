# GJXpress 小程序功能链路文档同步规则

## 规则说明

本规则用于强制要求：每次修改 miniprogram 功能时，必须同步更新 `miniprogram/docs/feature-flow-acceptance.md`。

## 触发条件

当 Windsurf 执行以下任何操作时，必须检查并更新 feature-flow-acceptance.md：

1. **页面变更**
   - 新增页面到 `miniprogram/pages/`
   - 删除页面
   - 重命名页面路径
   - 修改 `app.json` 中的页面配置

2. **TabBar/导航变更**
   - 修改 `app.json` 中的 `tabBar` 配置
   - 修改页面跳转逻辑
   - 新增/删除导航入口

3. **API 变更**
   - 新增 API 调用
   - 删除 API 调用
   - 修改 API 路径或参数
   - 修改 `request.js` 逻辑

4. **状态变更**
   - 修改订单状态定义
   - 修改包裹状态定义
   - 修改支付状态定义
   - 修改 `utils/status.js`

5. **用户交互变更**
   - 新增按钮
   - 修改按钮功能
   - 新增用户输入表单
   - 修改弹窗/提示逻辑

6. **登录鉴权变更**
   - 修改登录流程
   - 修改 token 处理
   - 修改 `utils/auth.js`

7. **图片/文件变更**
   - 新增图片展示逻辑
   - 新增图片上传功能
   - 修改图片预览逻辑

8. **文案/审核相关变更**
   - 修改隐私政策
   - 修改客服相关内容
   - 修改任何微信审核敏感文案

## 强制检查清单

在提交任何 miniprogram 代码变更前，Windsurf 必须回答：

```
1. 是否新增/删除/修改了页面？
2. 是否新增/删除/修改了用户入口或按钮？
3. 是否新增/删除/修改了 API？
4. 是否改变了订单、包裹、支付、异常状态？
5. 是否改变了登录、token、request.js？
6. 是否改变了隐私政策、客服、审核相关文案？
7. 是否新增了用户输入或图片能力？
8. 是否涉及支付、费用、支付状态？
9. 是否需要新增验收用例？
10. 是否已更新 miniprogram/docs/feature-flow-acceptance.md？
```

## 更新要求

如果以上任何一项为"是"，必须：

1. **更新文档**：
   - 如果是新增页面，添加到"页面清单"章节
   - 如果是新增 API，添加到"API 使用矩阵"章节
   - 如果是新增用户链路，添加到"完整用户链路"章节
   - 如果是新增验收用例，添加到"页面级验收用例"章节
   - 如果涉及状态变更，更新"状态流转图"章节

2. **更新变更日志**：
   - 在文档末尾的"变更日志"表格中添加新记录

3. **标记不一致项（如需要）**：
   - 如果发现代码与文档不一致，添加到"不一致项、风险项、待确认项"章节

## 无需更新文档的情况

以下纯技术优化类变更，可在开发总结中说明"**本次无 feature-flow 文档更新需求**"：

- UI 样式调整（颜色、间距、字体）
- 代码重构（提取函数、重命名变量）
- 性能优化（减少重渲染、缓存）
- 注释完善
- 错误日志格式调整
- 修复纯 bug（不改变功能链路）

## 执行步骤

当检测到需要更新文档时，Windsurf 必须按以下顺序执行：

1. **读取当前文档**：`miniprogram/docs/feature-flow-acceptance.md`
2. **分析变更影响**：确定需要更新哪些章节
3. **保留原有内容**：不要删除已有章节，只追加或修改
4. **更新相关章节**：
   - 页面清单
   - API 使用矩阵
   - 用户链路
   - 验收用例
   - 变更日志
5. **检查风险项**：如有新增风险，添加到风险清单
6. **保存文档**
7. **在回复中确认**：明确说明"已同步更新 feature-flow-acceptance.md"

## 禁止行为

❌ **绝对禁止**：
- 只改代码，不更新文档
- 删除文档中的已有验收用例（除非功能已删除）
- 修改功能后不更新状态流转表
- 新增 API 后不更新 API 矩阵

## 示例回复格式

当完成代码和文档同步更新后，回复格式应为：

```
## 代码变更
- 修改了 pages/xxx/index.js，新增 xxx 功能
- 修改了 services/xxx.service.js，新增 xxx API 调用

## 文档同步更新
- ✅ 已更新 miniprogram/docs/feature-flow-acceptance.md
- 更新章节：
  - 3. 页面清单（新增 xxx 页面）
  - 7. API 使用矩阵（新增 POST /xxx）
  - 8. 页面级验收用例（新增 TEST-xxx）
  - 14. 变更日志（新增 v1.0.1 记录）
```

如果确认无需更新文档：

```
## 代码变更
- 优化了 xxx 页面的样式

## 文档同步检查
- 本次为纯样式优化，不影响功能链路
- **本次无 feature-flow 文档更新需求**
```
