# 广骏供应链服务微信小程序 PRD

> Project: GJXpress / 广骏供应链服务  
> Scope: WeChat Mini Program only  
> Backend: `https://api.gjxpress.net`  
> Status: MVP implementation document for Windsurf

---

## 1. 产品定位

微信小程序是「广骏供应链服务」面向客户的轻量入口。它不是完整网站、不是 Admin 后台，也不处理站内支付。它的核心目标是让客户在微信里完成物流服务链路中的关键动作：

1. 微信登录并绑定系统用户。
2. 复制国内仓收货地址。
3. 查看订单和包裹状态。
4. 查看国内仓入库照片、重量、尺寸。
5. 对入库包裹进行确认或提交异常。
6. 查看支付状态、出货信息和国际物流状态。
7. 后期接收入库、待确认、待支付、已发货等订阅消息。

小程序只调用后端 API：

```text
miniprogram → https://api.gjxpress.net → NestJS backend → Supabase Postgres / Storage
```

小程序不得直接调用 Supabase、Vercel Admin、Vercel SEO 网站或任何第三方物流 API。

## 3. 用户角色

### 3.1 普通用户

小程序只有普通用户角色。用户通过微信登录后获得系统内 `user_code`，用于标识仓库收货地址和订单归属。

用户能力：

- 登录。
- 查看个人 `user_code`。
- 复制国内仓地址。
- 查看订单列表。
- 查看订单详情。
- 查看包裹入库照片。
- 确认包裹无误。
- 提交包裹异常。
- 查看支付状态。
- 查看物流渠道、运单号和物流状态。
- 授权订阅消息。

### 3.2 管理员不在小程序内操作

管理员操作全部在 Admin 后台完成，不在小程序里提供后台功能。

小程序不得暴露：

- 入库录入。
- 支付状态修改。
- 强制发货。
- 订单状态 override。
- 操作日志管理。
- 系统配置。

## 4. 非目标范围

MVP 小程序不做：

- 微信支付。
- 支付宝支付。

- WebView 跳转到 `www.gjxpress.net` 或 `admin.gjxpress.net`。

- 直接访问第三方物流 API。

## 5. 核心业务流程

### 5.1 用户获取地址并下单

```text
用户打开小程序
→ 微信登录
→ 进入地址页
→ 复制国内仓地址
→ 在淘宝/京东等平台下单
→ 商品发往国内仓
```

### 5.2 国内仓入库后用户确认

```text
管理员在后台录入入库
→ 上传外包装 / 面单 / 内部照片
→ 系统生成包裹记录
→ 用户在小程序订单详情查看入库信息
→ 用户点击「确认无误」或「有问题」
```

### 5.3 支付状态与发货

```text
管理员审核订单
→ 设置待支付
→ 用户线下/外部完成支付
→ 管理员后台标记已支付
→ 管理员安排发货
→ 用户在小程序查看物流状态
```

小程序只展示支付状态，不发起支付。

### 5.4 收货确认 / 二维码

二维码功能为后续阶段。MVP 可先预留入口和 API wrapper，不强制实现完整交互。

## 6. 状态体系

### 6.1 Order Status

| 后端枚举 | 中文展示 | 用户说明 |
|---|---|---|
| `UNINBOUND` | 未入库 | 仓库暂未收到包裹 |
| `INBOUNDED` | 已入库 | 仓库已收到，等待进一步处理 |
| `USER_CONFIRM_PENDING` | 待用户确认 | 请查看照片并确认是否无误 |
| `REVIEW_PENDING` | 待审核 | 已确认，等待管理员审核 |
| `PAYMENT_PENDING` | 待支付 | 请根据客服或后台通知完成支付 |
| `PAID` | 已支付 | 支付已确认，等待安排出货 |
| `READY_TO_SHIP` | 待发货 | 已进入出货准备 |
| `SHIPPED` | 已发货 | 国际物流已发出 |
| `COMPLETED` | 已完成 | 订单完成 |
| `EXCEPTION` | 异常处理中 | 订单存在异常，等待人工处理 |

### 6.2 Package Status

| 后端枚举 | 中文展示 | 用户动作 |
|---|---|---|
| `UNINBOUND` | 未入库 | 无 |
| `INBOUNDED` | 已入库 | 查看信息 |
| `USER_CONFIRM_PENDING` | 待确认 | 显示「确认无误 / 有问题」 |
| `CONFIRMED` | 已确认 | 无 |
| `EXCEPTION` | 异常处理中 | 查看异常状态 |
| `SHIPPED` | 已发货 | 查看物流 |

### 6.3 Payment Status

| 后端枚举 | 中文展示 | 用户说明 |
|---|---|---|
| `UNPAID` | 未支付 | 订单暂不进入发货流程 |
| `PROCESSING` | 支付处理中 | 等待管理员确认 |
| `PAID` | 已支付 | 可进入发货流程 |

## 7. 核心功能模块

## 7.1 Auth 登录模块

### 目标

通过微信登录建立系统用户身份。

### 流程

```text
wx.login()
→ 获得 code
→ POST /auth/login
→ 后端 code2Session 获取 openid
→ 后端创建或更新 User
→ 后端返回 JWT + user profile
→ 小程序保存 token
```

### 小程序端要求

- 不保存 `openid` 作为权限判断依据。
- 不保存 AppSecret。
- 不把 AppSecret、Supabase key、JWT secret 写到小程序代码。
- 后续请求都带：

```http
Authorization: Bearer <token>
```

### 用户信息策略

MVP 不强制获取头像和昵称。用户头像、昵称可以为空。

如果未来需要头像昵称，优先使用微信当前推荐的用户主动选择能力：

- `button open-type="chooseAvatar"` 获取头像。
- `input type="nickname"` 获取昵称。

不得在用户无感知情况下强制收集头像昵称。

---

## 7.2 地址管理模块

### 目标

给用户提供国内仓地址，并支持一键复制，方便用户在电商平台下单。

### 地址格式

建议完整复制内容：

```text
收件人：广骏仓-{{user_code}}
电话：{{warehouse_phone}}
地址：{{warehouse_full_address}}
备注：请保留用户ID {{user_code}}
```

### 交互

- 点击「一键复制完整地址」调用 `wx.setClipboardData`。
- 复制成功后显示 Toast：`已复制，请粘贴到淘宝/京东收货地址`。
- 如果用户未登录，先登录，再展示地址。

### 数据来源

地址信息由后端接口返回，避免小程序硬编码：

```http
GET /public/warehouse-address
```

如果后端暂未实现，可临时在 `config.js` 中放 mock address，但上线前应改为接口返回。

## 7.4 订单详情模块

### 目标

展示订单全貌，包括状态、包裹、费用、物流。

### API

```http
GET /orders/:id
```

### 展示区域

1. 订单状态卡。
2. 支付状态卡。
3. 包裹列表。
4. 入库照片。
5. 重量与体积信息。
6. 费用信息。
7. 物流信息。
8. 异常信息。

### 权限

用户只能查看属于自己的订单。权限由后端校验。

如果后端返回 403，小程序显示：

```text
无权查看该订单
```

如果后端返回 404，小程序显示：

```text
订单不存在或已删除
```

## 7.5 包裹确认模块

### 目标

当包裹状态为 `USER_CONFIRM_PENDING` 时，让用户确认包裹无误或提交异常。

### 确认无误

按钮：

```text
确认无误
```

点击后弹出二次确认：

```text
确认后将进入下一步审核流程，是否确认？
```

确认后调用：

```http
POST /packages/:id/confirm
```

成功提示：

```text
已确认，等待审核
```

### 有问题

按钮：

```text
有问题
```

点击后弹窗输入问题描述。

调用：

```http
POST /packages/:id/issue
```

请求体：

```json
{
  "type": "OTHER",
  "description": "用户填写的问题描述"
}
```

成功提示：

```text
已提交异常，客服会尽快处理
```

## 7.6 图片展示模块

### 图片类型

| 类型 | 中文 |
|---|---|
| `OUTER` | 外包装 |
| `LABEL` | 面单 |
| `INNER` | 内部物品 |
| `EXCEPTION` | 异常照片 |

### 显示要求

- 每个包裹至少展示外包装、面单图片。
- 图片以缩略图形式展示。
- 点击图片调用 `wx.previewImage`。
- 图片加载失败时显示占位图和文字：`图片加载失败`。

### URL 策略

MVP 建议后端返回 `api.gjxpress.net` 域名下的图片代理 URL，例如：

```text
https://api.gjxpress.net/files/{imageId}
```

这样小程序只需要配置 `api.gjxpress.net` 作为 downloadFile 合法域名，不需要直接配置 Supabase Storage 域名。

## 7.7 通知订阅模块

### 目标

后期用于入库、待确认、待支付、已发货等提醒。

### MVP 策略

MVP 可以先实现 UI 入口和授权调用，后端发送逻辑可在后续补齐。

触发点：

- 首页「订阅物流提醒」。
- 订单详情「订阅此订单提醒」。
- 包裹确认成功后提示订阅。

### 模板建议

需要在微信公众平台订阅消息中申请类似模板：

- 包裹入库提醒。
- 待确认提醒。
- 支付状态提醒。
- 发货提醒。

小程序调用：

```js
wx.requestSubscribeMessage({
  tmplIds: [TEMPLATE_ID_1, TEMPLATE_ID_2],
  success(res) {},
  fail(err) {}
})
```

注意：必须由用户点击行为触发，不要在 `onLoad` 自动弹出。

## 8. 隐私与合规

### 8.1 需要声明的信息

小程序隐私保护指引应覆盖：

- 微信用户标识，用于识别用户身份。
- 订单信息，用于展示用户物流记录。
- 包裹照片，用于入库确认和物流服务。
- 用户异常反馈内容，用于客服处理。
- 用户联系方式，如未来需要客服回访。

### 8.2 隐私交互

在以下行为前检查用户隐私授权：

- 登录。
- 查看订单。
- 查看包裹照片。
- 提交异常。
- 订阅消息。

### 8.3 不采集不必要信息

MVP 不采集：

- 身份证。
- 人脸。
- 银行卡。
- 精确定位。
- 通讯录。

## 9. 错误处理要求

### 9.1 网络错误

显示：

```text
网络连接失败，请稍后重试
```

### 9.2 登录失败

显示：

```text
登录失败，请重新进入小程序或联系客服
```

### 9.3 Token 过期

处理：

```text
收到 401 → 清除 token → 重新 wx.login → 重试一次当前请求
```

如果仍失败，提示用户重新打开小程序。

### 9.4 API 结构错误

记录 console error，页面显示：

```text
数据加载失败，请稍后重试
```


建议后端支持为审核用户返回一条 mock/demo 订单，或在空状态中清晰展示服务用途。

## 11. 验收标准

### 11.1 登录

- 真机能调用 `wx.login`。
- 后端能创建真实微信用户。
- 小程序能保存 JWT。
- 后续请求带 token。

### 11.2 地址

- 能展示用户 `user_code`。
- 能复制完整仓库地址。

### 11.3 订单

- 能加载订单列表。
- 能进入订单详情。
- 状态文案正确。
- 空状态正确。

### 11.4 包裹确认

- `USER_CONFIRM_PENDING` 包裹显示两个操作按钮。
- 确认成功后状态刷新。
- 异常提交成功后进入异常状态。

### 11.5 图片

- 图片能展示。
- 点击可预览。
- 加载失败有 fallback。

### 11.6 审核

- 页面无敏感词。
- 隐私入口可访问。
- 不出现未实现按钮。
- 小程序只调用 `api.gjxpress.net`。
- 不出现 Admin 页面入口。
